Introduction to Node.js
    Node.js is an open-source, cross-platform runtime environment that allows developers to execute javascript code outside of a web browser.

    It was developed by Ryan dahl in 2009 and it is built on google chromme's V8 javascript engine.

    Traditional javascript, which is primary used for client-side scripting in web browsers, Node.js enables developers to use javascript for server - side programming . allowing for the creation of scalable network applications.

Key features of node.js :
    1. Asynchronous and event driven
    2. Single - threaded but highly scalable 
    3. Fast and lightweight
    4. Rich ecosystem with npm
    5. cross platform(can use on any os like windows,linus,mac)

common use cases for Node.Js:
    1. web applicatiom development 
    2. API services
    3. Real time applications
    4. Microservices
    5. Serverless computing

Overall Node.js popular for its performance, scalability and large ecosystem, making it a perfect chioce for developers building modern, data - intensive application.


Key components of Node.js:
    a. Single-threaded, event-driven architecture:
        - Single - Threaded, normally, many programming environments use multiple threads (or workers) to handle tasks, especially when doing lots of things at once. Node.js however , uses just one thread. It can do this because it relies on something called the event loop to manage multiple tasks without needing multiple threades.

        - Event - Driven: Think of Node.js as someone who is great at multitasking. Instead of waiting for one task to finish , Node.js can start a task, move on to the next one, and check back on the first task when it's done. This is all thans to its event driven nature.

    b. Event Loop:
        - The event loop is what makes Node.js able to multitask. Here's how it works:
            1. Timers: Handles scheduled tasks, like setTimeout and setInterval
            2. Pending Callbacks: Deals with tasks that are delayed until the next loop.
            3. Poll: Listens for new tasks, such as reading from files or receiving data from the internet. 
            4. check: Run callbacks from setimmediate
            5. close callbacks: Handles events like closing a file or network connection.
    
    c. Non - Blocking I/O:
        - Non - blocking means Node.js doesn't wait for tasks to finish before moving on to the next one. for example, if node.js asks for data from a database, it won't sit and wait for that data. instead , it'll move on and handle other tasks until the data is ready.
        - This makes node.js very efficient at handling many requests at once. Rather than pausing, it's conatantly moving from task to task, which helps make your application faster and more responsive.

    d. V8 javascript Engine:
        - Node.js is built on V8, the same engine that powers google chrome. v8 takes javascript code and quickly turns it into machine code that your computer can understand. This makes running javascript javascript really fast. 
        - Garbage collection: V8 also automatically cleans up unused memory, so you don't have to worry about your application running out of memory as it runs. 
        - Just - in - Time (JIT) compilation: v8 compiles javascript on the fly-while your app runs, which helps make it even faster.

Node.js Modules:
    in node.js modules refer to javascript files that contain reusable pieces of code, such as middleware, routes, controllers, utilities and configurations. These modules help organize an application into smaller, managable chunks, making the codebase more moduler, readable and maintainable.

    - Types of Modules in Node.js:
        1. Core Modules (http, fs, path, etc.)
        2. Third party Modules(express, dotenv, body-parser etc)
        3. custom modules(): we can create custom modules to reuse a piece of code multiple times in multiple files that will help us to make codebase more readable and moduler. 

    Node.js provides a set of core modules that allow you to perform essential tasks such as 
        - Handling files 
        - Creating web servers 
        - Working with file paths

    These modules are built into Node.js so don't need to install them separately. Let's g through some of the commonly used core modules with beginner - friendly examples.

        1. file system(fs): by using file we can read, write, and delete the files and directory in our system     
        2. HTTP: Allow us to create web server that can listen to client request and send the response
        3. Path: help us to work with file, directory in a way that is compatible across different operating systems.
        4. OS: provides informations about the operating systems such as cpu architecture, free memory, total memory and system uptimes etc.
        5. URL: Allow us to parse and construct the url making it easy to handle web addresses.
        6. Crypto: provide crypto graphy functionality such as generating hashes or encripting the data.
