# scratch-to-scale

> the long way from `console.log("hello world")` to systems that survive contact with real users

---

I'm Krish. CS student, currently spending most of my time around AI, the MERN stack, and data science — but full stack web dev is the foundation I keep coming back to, so I'm rebuilding it properly instead of relying on whatever stuck from random tutorials over the years.

This repo is the record of that process. Not a "100 days of code" streak that quietly dies in week two — a roadmap with actual checkpoints, where every phase ends in something real and deployed, not just a tutorial I followed along with.

If you're doing something similar, take whatever's useful here. If you're a recruiter skimming this — hi — the progress tracker below is the fastest way to see exactly where I am right now.

## Why "scratch to scale"

Most learning repos stop at "I built a CRUD app." The harder, more interesting part is everything after that — auth that doesn't leak data, APIs that don't fall over under load, deployments that don't need a prayer before pushing to prod. That's the actual target here: production-level, not tutorial-level.

## Roadmap

### Phase 0 — Foundations

- [ ] HTML5 — semantics, accessibility, forms
- [ ] CSS — flexbox, grid, responsive design, animations
- [ ] JavaScript — ES6+, closures, async/await, the event loop
- [ ] Git & GitHub — branching, PRs, resolving conflicts like an adult

### Phase 1 — Frontend Engineering

- [ ] React — hooks, context, component patterns
- [ ] TypeScript
- [ ] State management (Redux / Zustand)
- [ ] Testing — Jest, React Testing Library
- [ ] Build tooling — Vite, bundling, performance basics

### Phase 2 — Backend Engineering

- [ ] Node.js & Express fundamentals
- [ ] REST API design (and when REST isn't the right call)
- [ ] Auth — JWT, OAuth, sessions, refresh tokens
- [ ] Databases — MongoDB & PostgreSQL, schema design
- [ ] ORMs — Mongoose / Prisma

### Phase 3 — Production Readiness

- [ ] Docker & containerization
- [ ] CI/CD pipelines (GitHub Actions)
- [ ] Cloud deployment — AWS / Vercel / Render
- [ ] Caching, rate limiting, structured logging
- [ ] Security basics — OWASP top 10, input validation, secrets management

### Phase 4 — Scale & Systems

- [ ] System design fundamentals
- [ ] Microservices vs. monoliths — when each actually makes sense
- [ ] Message queues / event-driven patterns
- [ ] Load balancing & horizontal scaling
- [ ] Observability — metrics, tracing, alerting
